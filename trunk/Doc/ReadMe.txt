++ Общее описание работы системы

Система SAV Access позволяет организовать, в малых и средних корпоративных ЛВС, систему дистрибуции (распространения) программного обеспечения и файлов данных на клиентские компьютеры. Система имеет механизм Plugin API позволяющий разработать плагины, и расширить функционал системы до необходимого уровня.

Система интегрируется с имеющейся в корпоративной ЛВС архитектурой Microsoft Active Directory.

Основной алгоритм работы системы:

Администратор системы с помощью приложения SAV Administrator создает на доступном пользователям системы для чтения корпоративном сетевом ресурсе Windows (SMB) сетевое хранилище, сохраняя информацию о конфигурации в доступном для всех пользователей системы файле \*.savaccess.

В созданном хранилище администратор системы в первую очередь добавляет домен Active Directory, далее в зависимости от необходимости добавляет доменные группы, либо создает собственные группы пользователей системы, добавляет пользователей системы (определяются из домена).

Далее администратор определяет правила дистрибуции(см. ниже) для всех добавленных контейнеров системы (домен, группы AD, собственные группы, пользователи). Также с помощью служебной утилиты создается файл контроля доступа (список менеджеров). Файл контроля доступа к системе содержит список пользователей и соответствующие им контрольные суммы паролей. Файл служит для повышения безопасности удаленног доступа к клиентским модулям. Расположения файла доступа хранится в реестре и должно быть распространено по пользовательским компьютерам с помощью груповых политик Ms AD, либо иным способом (через импорт ключей реестра например).

На клиентских компьютерах первый раз клиентский модуль SAV User должен быть запущен либо с параметром - полным именем файла конфигурации \*.savaccess, либо с помощью запускающего консольного приложения SAV Starter (для работы SAV Starter на клиентских ПК должен быть распространено место нахождения файла контроля доступа, для варианта с запуском SAV User с параметром распространение информации о файле контроля доступа не является обязательным). Во время первого запуска на клиентском компьютере создаются необходимые файлы конфигурации в частности сохраняется полное имя файла конфигурации хранилища. При последующих запусках клиентского модуля данное хранилище будет считаться основным, правила определенные в данном хранилище будут применяться каждый раз при запуске клиентского модуля. 

При этом, используется следующий приоритет применения правил: правила домена -> правила групп MS AD -> правила собственный групп -> правила для пользователя.

+++ Правила дистрибуции

Правила представляют из себя какие-либо действия выполняемые на клиентском компьютере. По умолчанию (без дополнительных плагинов) стандартный набор доступных правил (действий) включает в себя операции по копированию, созданию и удалению директорий и файлов. 

Plugin API позволяет расширить доступные правила. Плагины в виде динамических библиотек с расширением ext поставляются совместно с клиентским приложением. 

Один плагин экспортирует одну функцию выполняющую одно действие над файлами(директориями, или иными объектами) определенного типа.

Например, на данный момент реализованы плагины:
    * [SAVPlugin/INIPlugin|Перезапись ini файла]
    * [SAVPlugin/TXTPlugin|Распаковка zip архива]
    * [SAVPlugin/ZIPPlugin|Дозапись текстового файла]
